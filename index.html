i2p KISS Search Engine 0.1

This is an early test. It may have huge bugs.
This search engine uses gatling, sh scripts for crawling and searching, wget, grep & sed, elinks for parsing html and plaintext as database


Search:


Known Problems:

  • No combination of search terms possible at the moment, multiple search terms only searches for an exact match of the phrase
  • Results can be badly truncated. I try to "html-ify" the pages with broad paragraphs but it doesn't always work
  • Due to searching in b32 AND normal urls there will often be duplicate results.
  • Layout with multiple results sucks for now
  • Composing the search results with variables in bash is slow.
